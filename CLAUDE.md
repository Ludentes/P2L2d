# portrait-to-live2d — Project Instructions

## Goal

Automated pipeline from a single portrait image → animatable Live2D `.moc3` model, driven by face tracking + BCI signals in real time. Based on the Textoon/CartoonAlive approach (HaiMeng rig template + generated textures + learned MediaPipe→parameter MLP).

## Stack

- **Python 3.12**, uv for package management
- **ComfyUI** (programmatic via REST API + MCP server) — texture generation, layer decomposition
- **Live2D Cubism SDK** (Python bindings via `live2d-py`) — model rendering, training data generation
- **MediaPipe** — face landmark extraction
- **PyTorch** — CartoonAlive MLP training
- **SDXL + ControlNet** — component texture generation
- **SAM2** — segmentation

## Key Concepts

- **HaiMeng dataset**: the base rig. 9 × 4096px texture sheets, 107 params, 24 ARKit blendshapes, 7 garment toggles. Behind EULA (submit at github.com/Human3DAIGC/Textoon) — academic-only outputs accepted.
- **Textoon pipeline**: template copy + texture swap. NOT auto-rigging. `utils/transfer_part_texture.py` handles UV crop/paste.
- **CartoonAlive MLP**: 4-layer network, 478 MediaPipe landmarks → 107 Live2D params. Trained on 100k synthetic renders. Code unreleased — implement from paper (arXiv:2507.17327).

## BCI Parameter Contract

Models generated by this pipeline must include these custom parameters so the Muse VTuber Bridge (ZyphraExps project) can drive them:

| Parameter ID | Range | Signal source |
|---|---|---|
| `ParamJawClench` | 0–1 | EMG jaw clench |
| `ParamFocusLevel` | 0–1 | EEG theta/beta concentration |
| `ParamRelaxation` | 0–1 | EEG alpha relaxation |
| `ParamHeartbeat` | 0–1 | PPG pulse |
